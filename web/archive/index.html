WSML2Reasoner Framework - Overview

Overview | Releases | Online Demo | Javadoc | View Source | Snapshot

The WSML2Reasoner Framework is a highly modular architecture which combines various validation, normalization, and transformation functionalities essential to the translation of ontology descriptions in WSML to the appropriate syntax of several underlying reasoning engines.

Quick release (including IRIS reasoning engine): wsml2reasoner-iris-lgpl-v0_5.zip.
Quick release (without underlying reasoning engine): wsml2reasoner-lgpl-v0_5.zip.

Generic (i.e., not including WSMO or reasoning engine libraries) source and binary distributions can be obtained from the snapshot directory (generated daily). Please note that this is a work in progress and should at best be considered alpha quality code. We are continually trying to improve the quality of the software. Comments and suggestions are welcome.

Dependencies

This software depends on WSMO4j. The required compatible WSMO jars (wsmo4j & wsmlparser) are included in the WSMO, Datalog, and DL releases (found in the lib dir).

This framework handles translations only; to actually perform reasoning you will need a reasoning engine. Wrappers are currently implemented and tested for:

  • IRIS (WSML-Flight)
  • MINS (WSML-Rule)
  • KAON2 (WSML-Flight)
  • PELLET (WSML-DL)

For using the DL support you will need additionally the OWL-API, as well as several other jars included in the release.
Seperate releases which include the necessary libraries are provided to cater to reasoning requirements and licensing issues. An explanation of these releases can be found here.
Brief Explanation

The essential end-user parts of the WSML 2 Reasoner Framework example are loading and registering the ontology file, and then posing a query.

  • In order to run the ReasonerExample (distributed with all source releases - found in 'src/example/'), you will need a sample WSML ontology file.
  • In order to load this ontology, change the path and filename in "Ontology exampleOntology = loadOntology("example/humanOntology.wsml");" to point to your example ontology file.
  • In order to pose a query, change the query in "String queryString = "?x memberOf ?y";" to your liking. Currently, this query returns all instances that are a member of a concept.
  • For further explanation of the WSMO object model and language, please refer to documentation and examples provided by the WSMO working group.
