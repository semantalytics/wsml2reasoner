WSML 2 Reasoner Explanation

The WSML 2 Reasoner framework can translate ontology descriptions in WSML to predicates and rules. Additionally it provides a facade for easy integration for different reasoners. This page provides the essential information for first time users.

Source and binary distributions can be obtained from the snapshot directory (generated daily).

KAON2 (WSML Flight), MINS (WSML Rule), and Pellet (WSML DL).
Required Libraries
  • Reasoning Engine (choose one):
      MINS -mins-XXXXXXXX.jar (the latest snap shot) - recommended for WSML Rule. The ReasonerExample.java is currently set to use MINS.
      KAON2 -kaon2.jar
  • The WSMO Libraries -
      wsmlparser-20060210.jar
      wsmo4j-0.5.2.jar
      wsmo-api-0.5.2.jar
    All of which can be found at WSMO releases in the wsmo4j-0.5.2\wsmo4j\lib directory.
Brief Explanation

The essential parts of the WSML 2 Reasoner example are loading and registering the ontology file and then posing a query.

  • In order to run the ReasonerExample, you will need a sample WSML ontology file.
  • In order to load this ontology, change the path and filename in "Ontology exampleOntology = loadOntology("example/humanOntology.wsml");" to point to your example ontology file
  • In order to pose a query, change the query in "String queryString = "?x memberOf ?y";" to your liking. Currently, this query returns all instances that are a member of a concept.
  • For further explanation of the WSMO object model and language, please refer to documentation and examples provided by the WSMO working group.

A simple Online Demo has been developed based up on this framework.
